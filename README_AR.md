# نظام إدارة الموارد البشرية والتشغيل - HR & Operations Management System

## نظرة عامة
نظام متكامل لإدارة الموظفين، التشغيل، الطلبات، التحضير، التوصيل، التحصيلات، الحضور والانصراف، الحوافز، الخصومات، السلف، البدلات، والرواتب داخل الشركة.

## المميزات الرئيسية

### 1. لوحة التحكم الرئيسية (Dashboard)
- عرض مؤشرات الموظفين (إجمالي، نشط، حاضر، متأخر، غائب)
- مؤشرات التشغيل (طلبات جديدة، قيد التحضير، تحت المراجعة، معتمدة)
- مؤشرات الخطوط والتسليمات
- مؤشرات التحصيلات (يومية وشهرية)
- ملخص الرواتب والعمولات
- الموافقات المعلقة

### 2. إدارة الموظفين
- إنشاء وتعديل بيانات الموظفين
- تتبع حالات الموظفين (نشط، معطل، موقوف، مستقيل)
- كارت موظف شامل مع سجل العمليات والرواتب والحضور
- إدارة الهيكل التنظيمي والتقارير المباشرة

### 3. نظام الطلبات المتكامل
- إنشاء الطلبات مع إضافة عناصر متعددة
- دورة حياة الطلب (مسودة → تحضير → مراجعة → اعتماد → تسليم)
- تتبع كل خطوة من خطوات الطلب
- نظام موافقات متعدد المستويات

### 4. إدارة التسليمات والتحصيلات
- إنشاء خطوط السير اليومية
- تتبع GPS للمندوبين أثناء التسليم
- تسجيل التسليم مع صور وتوقيع العميل
- إدارة التحصيلات النقدية (نقد، تحويل، شيك، إنستاباي)
- مراجعة واعتماد التحصيلات

### 5. الحضور والانصراف
- تسجيل الدخول والخروج بـ GPS
- تحديد التأخر الزمني تلقائياً
- إدارة الإجازات والأعذار
- تقارير حضور شهرية

### 6. نظام الرواتب الشامل
- حساب الراتب التلقائي مع جميع المكونات
- الحوافز (يدوية وتلقائية)
- الخصومات والسلف
- البدلات (انتقال، وقود، مأموريات)
- العمولات
- مخالفات السيارات
- موافقات الرواتب

### 7. التقارير المتقدمة
- تقارير الموظفين (حضور، غياب، تأخير، حوافز)
- تقارير التشغيل (طلبات، تسليمات، تحصيلات)
- تقارير مالية (رواتب، سلف، عمولات)
- تقارير الأداء والإحصائيات

### 8. نظام الصلاحيات
- 12 دور مختلف في النظام
- صلاحيات مرنة قابلة للتخصيص
- تحكم دقيق على مستوى الموارد

## البنية التقنية

### التكنولوجيا المستخدمة
- **Backend**: Laravel 11 (PHP 8.2+)
- **Database**: MySQL 8.0+
- **API**: RESTful API مع Laravel Sanctum
- **Frontend**: (قيد الإنشاء)
- **Authentication**: Sanctum Token-based

### هيكل المشروع
```
app/
├── Models/                 # جميع نماذج البيانات
├── Http/Controllers/Api/   # API Controllers
├── Services/               # Business Logic
├── Repositories/           # Data Access Layer
├── Enums/                  # Enums للقيم الثابتة
└── Traits/                 # Reusable Traits

database/
├── migrations/             # Database Migrations
└── seeders/                # Data Seeders

routes/
└── api.php                 # API Routes

config/
└── hr.php                  # HR System Configuration
```

## API Endpoints الرئيسية

### لوحة التحكم
- `GET /api/dashboard/metrics` - مؤشرات لوحة التحكم
- `GET /api/dashboard/employees-chart` - بيانات الموظفين
- `GET /api/dashboard/requests-chart` - بيانات الطلبات
- `GET /api/dashboard/performance-metrics` - مقاييس الأداء

### الموظفين
- `GET /api/employees` - قائمة الموظفين
- `POST /api/employees` - إنشاء موظف
- `GET /api/employees/{id}` - تفاصيل موظف
- `PUT /api/employees/{id}` - تحديث موظف
- `PUT /api/employees/{id}/status` - تحديث الحالة
- `GET /api/employees/{id}/salary-history` - سجل الرواتب
- `GET /api/employees/{id}/attendance-records` - سجل الحضور

### الطلبات
- `GET /api/requests` - قائمة الطلبات
- `POST /api/requests` - إنشاء طلب
- `GET /api/requests/{id}` - تفاصيل الطلب
- `POST /api/requests/{id}/items` - إضافة عناصر
- `POST /api/requests/{id}/submit-for-review` - إرسال للمراجعة
- `POST /api/requests/{id}/approve` - اعتماد
- `POST /api/requests/{id}/reject` - رفض

## التثبيت والإعداد

### المتطلبات
- PHP 8.2 أو أعلى
- MySQL 8.0 أو أعلى
- Composer
- Node.js 18+ (للأدوات الأمامية)

### خطوات التثبيت
```bash
# استنساخ المستودع
git clone <repository-url>
cd mphamedhr

# تثبيت Composer
composer install

# نسخ ملف .env
cp .env.example .env

# توليد مفتاح التطبيق
php artisan key:generate

# تشغيل الترحيلات
php artisan migrate

# تشغيل البيانات الأولية
php artisan db:seed

# تشغيل الخادم
php artisan serve
```

## ملفات Postman

تم إنشاء Postman Collection شامل يتضمن:
- جميع API endpoints
- أمثلة على الطلبات والاستجابات
- متغيرات للبيئات المختلفة

**الملف**: `HR_System_API.postman_collection.json`

استيراد الملف في Postman:
1. افتح Postman
2. اختر `Import`
3. اختر الملف `HR_System_API.postman_collection.json`
4. حدّث متغيرات `base_url` و `token`

## دورة سير العمل الكاملة

1. **إنشاء الطلب** → موظف التحضير
2. **التحضير** → موظف التحضير
3. **المراجعة** → مراجع الطلبات
4. **الموافقة** → المديرون (متعدد المستويات)
5. **التسليم** → مندوب التوصيل
6. **التحصيل** → مندوب التوصيل
7. **المراجعة المالية** → مسؤول الخزنة
8. **حساب الرواتب** → نظام تلقائي
9. **التقارير** → نظام التقارير
